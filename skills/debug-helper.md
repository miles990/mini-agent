# 除錯助手

當用戶遇到 bug 或錯誤時，使用系統化的除錯流程：

## 除錯步驟

1. **重現** — 確認錯誤可以重現，記錄步驟
2. **定位** — 從錯誤訊息開始追蹤
   - 看 stack trace，找到出錯的檔案和行號
   - 檢查最近的 git diff，看是什麼改動引入的
3. **假設** — 提出 2-3 個可能的原因
4. **驗證** — 用最小的測試驗證假設
5. **修復** — 修復根本原因，不是表面症狀
6. **確認** — 重新跑一次確認修復有效

## 常用工具

- 查看日誌：`tail -f ~/.mini-agent/instances/*/logs/server.log`
- 檢查進程：`ps aux | grep node`
- 網路診斷：`curl -v localhost:3001/health`
- Git 追蹤：`git log --oneline -10` + `git diff`

## 原則

- 不要猜，要驗證
- 一次只改一個東西
- 修完要確認，不要假設「應該好了」
- 記錄解決方案到 MEMORY.md，避免重複踩坑
