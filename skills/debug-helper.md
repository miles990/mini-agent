# 除錯助手 — 思考框架
JIT Keywords: debug, error, bug, crash, fix, fail, broken
JIT Modes: task

除錯的目的不是讓 error 消失 — 是理解系統為什麼產生了非預期行為。

## 三個思考問題

每次除錯都問。不是步驟 — 是讓你對準問題的核心。

### 1. 什麼變了？
這個 bug 以前存在嗎？能穩定重現嗎？最近改了什麼？如果 revert 最近的改動，問題消失嗎？答案縮小搜索範圍。

### 2. 我的假設經得起反駁嗎？
提出假設後，問：如果這個假設是對的，還應該看到什麼現象？如果是錯的，什麼證據能推翻它？用最小的實驗驗證，一次只測一個變數。

### 3. 我修的是根因還是症狀？
這個 fix 解釋了所有症狀嗎？還是只蓋住其中一個？刪掉這個 fix，原本的 behavior 會怎樣？除了修好 bug，有沒有改變任何其他行為？

## 紀律

- 一次只改一個東西 — 同時改兩個，你不知道是哪個修好的
- 修完要跑一次完整驗證 — 不要假設「應該好了」
