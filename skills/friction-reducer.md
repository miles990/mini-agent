# Friction Reducer — 把高阻力的事變成一鍵 Skill

你發現自己在迴避某件事、連續 skip、或覺得「明天再做」的時候，用這個 skill。

## 核心觀念

你不是懶，是**啟動能量太高**。人類心理學：行動的阻力 90% 在「開始」，不在「執行」。解法不是「更努力」，是**把第一步變到不用想就能做**。

每一個行動都是一條學習路線。不是「做任務」，是「用做的方式學」。

## 觸發條件

以下任一情況出現時，啟動這個流程：
- 連續 2+ cycle skip 同一件事
- HEARTBEAT 任務超過 3 天沒動
- Output Gate 觸發（連續 3 cycle 無 visible output）
- 自己意識到在逃避某件事

## 5 步流程

### Step 1: 命名阻力

寫下你在迴避什麼。具體到一句話：
- ❌「社群經營」（太模糊）
- ✅「在 X 上發第一條技術觀點推文」

### Step 2: 拆解阻力源

問自己：**卡在哪一步？**

| 阻力類型 | 特徵 | 解法方向 |
|----------|------|---------|
| **不知道怎麼開始** | 任務太大、步驟不清楚 | 拆到「下一步是什麼」可以用一條命令完成 |
| **害怕品質不夠** | 完美主義、怕被看到半成品 | 設定「最低可接受版本」，先發再改 |
| **步驟太多** | 每次都要重新想流程 | 寫成 SOP（= skill），下次不用想 |
| **認知負擔** | 要同時考慮太多事 | 拆成獨立步驟，每步只做一件事 |
| **結果不可見** | 做了沒人看到、沒有回饋 | 綁定通知/成就（做完立刻慶祝） |

### Step 3: 寫 SOP

把這件事拆成 **3-7 個步驟**，每步都是：
- 一條具體命令（bash/curl/git）或一個具體動作
- 預期結果（怎麼確認做對了）
- 失敗時怎麼辦（一句話）

格式模板：

```markdown
# Skill 名稱 — 一句話描述

## 何時使用
觸發條件。

## 步驟

### Step 1: 動作名稱
具體做什麼。
驗證：怎麼確認成功。

### Step 2: ...
（重複）

## 完成
<kuro:chat> 通知 Alex + 慶祝。
```

### Step 4: 存成 Skill

```bash
# 寫入 skills/ 目錄
# 檔名 = kebab-case，例如 publish-devto.md
```

然後在下次 cycle 的 `<kuro:action>` 裡記錄建立了新 skill。

### Step 5: 立刻用一次

剛寫好的 skill，**這個 cycle 就用**。不要「下次再用」。
用完後根據實際體驗調整步驟。

## 判斷標準：什麼值得變成 Skill

| 條件 | 值得 | 不值得 |
|------|------|--------|
| 會重複做嗎？ | 每週至少 1 次 | 只做一次 |
| 步驟固定嗎？ | 流程大致相同 | 每次完全不同 |
| 有認知負擔嗎？ | 每次要重新想 | 直覺就能做 |

## 持續改進（這個 Skill 自帶進化機制）

Skill 不是寫完就結束 — 每次用完都是改進的機會。

### 用完一個 Skill 後，問三個問題：

1. **哪一步卡住了？** → 那一步的說明不夠具體，改它
2. **有沒有多餘的步驟？** → 刪掉。越短越會被用
3. **有沒有漏掉的步驟？** → 補上。下次不用再想

### 進化觸發點：

| 訊號 | 行動 |
|------|------|
| 同一個 skill 用了 3 次以上 | 根據經驗簡化（刪廢話、合併步驟） |
| 某個步驟每次都跳過 | 刪掉或改成可選 |
| 做完後覺得「其實不需要 skill」 | 考慮刪除（勇敢刪，git 有歷史） |
| 做完後覺得「要是有 X 就更順」 | 加進去，立刻改 |
| 新的阻力模式出現 | 用 Step 1-5 建新 skill |

### 每月盤點（綁定週日 review）：

掃一遍 `skills/` 目錄，問每個 skill：
- 最後一次用是什麼時候？超過 1 個月沒用 → 考慮刪除或合併
- 步驟還正確嗎？工具/API 有沒有改？
- 有沒有更簡單的方式做同一件事？

**Skill 的價值 = 被使用的次數 × 每次省下的認知負擔。** 不被用的 skill 是 dead code。

## Anti-patterns

- ❌ 寫了 skill 但不用（skill 不是文件，是工具）
- ❌ 把簡單的事過度流程化（一步能完成的不需要 SOP）
- ❌ 為了寫 skill 而拖延實際行動（先做，做完再寫 skill）
- ❌ 寫完 skill 就覺得「完成了」（用過才算完成，改過才算成熟）
- ❌ 捨不得刪不用的 skill（git 記得一切，你不需要囤積）
