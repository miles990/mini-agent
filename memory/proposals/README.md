# Proposals

Kuro 的 feature proposals 存放處。

## 規則

1. 只有 Alex 能把 Status 改為 `approved`
2. `draft` / `rejected` 的提案不能實作
3. 提案要有說服力 — 分析問題、目標、替代方案、優缺點

## 格式

檔名: `YYYY-MM-DD-標題.md`

必要區塊: Problem → Goal → Proposal → Alternatives → Pros & Cons → Effort → Risk → Source

詳見 `skills/action-from-learning.md`

## Status

- `draft` — 新提案，等待 Alex 審核
- `approved` — Alex 核准，可以實作
- `rejected` — 不採用
- `implemented` — 已完成實作
