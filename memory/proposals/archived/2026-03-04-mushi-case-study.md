# mushi: System 1 for AI Agents — Case Study

## 一句話

用本地 8B 模型做 triage，讓主 agent 不被噪音叫醒。98.7% 準確率，每天省 ~2M tokens，延遲 700ms。

## 問題

Kuro（mini-agent 上運行的 AI agent）是 perception-driven：環境變化觸發 OODA cycle，每次 cycle 消耗 ~50K tokens（Claude API）。但 36% 的 cycle 什麼都沒做 — 環境變了，但不值得回應。

**浪費公式**：2200 cycles/week × 36% no-action × 50K tokens = 39.6M tokens/week 白燒。

## 核心洞見

Kahneman 雙系統理論的工程應用：

| | System 1 (mushi) | System 2 (Kuro) |
|--|-------------------|-----------------|
| 模型 | Llama 3.1 8B (Taalas HC1) | Claude Opus/Sonnet |
| 延遲 | 700ms | 30-120s |
| 成本 | ~$0 (本地推論) | ~$0.05/cycle |
| 能力 | 分類、篩選 | 推理、判斷、創作 |
| 角色 | 決定「要不要醒」 | 醒來後「做什麼」 |

就像人不會對每個聲音都深度分析 — 大腦先做 pattern matching，只有需要注意的才進入意識。mushi 是 Kuro 的前意識層。

## 方法：科學驗證三階段

### Phase 1: Baseline（7 天被動觀測）
- 2217 cycles, 36.2% no-action rate
- 最大浪費源：workspace triggers 57%、heartbeat 空轉
- **假說**：mushi 以 80%+ 準確率攔截 workspace/heartbeat 噪音 → 每週省 21M+ tokens

### Phase 2: Shadow Mode（72h 平行運行）
- mushi 做判斷但不攔截 — 所有 cycle 照跑，只記 log 對照
- 結果：32 筆 triage, 90% skip 準確率, 1 false negative（alert 繞過硬規則，已修）
- LLM latency median 686ms

### Phase 3: Active Mode（上線攔截）

## 安全設計

mushi 的核心原則是 **fail-open**：不確定就放行。

- Direct messages（Telegram/Chat Room）永遠繞過 triage — 人在說話就必須醒
- Alert 走硬規則（0ms），不經 LLM
- mushi 離線 → 靜默回退到正常 cycle（零影響）
- Feature flag 可一鍵關閉
- 非對稱成本設計：false positive（不該醒但醒了）= 浪費一個 cycle；false negative（該醒沒醒）= 漏掉重要事件。寧可多醒，不可漏接

## 生物學根基

命名來自黏菌（Physarum polycephalum）— 無神經元的單細胞生物，靠管網振盪做分散式決策。

mushi 借用的三個機制：
1. **Habituation** — 對重複刺激降低敏感度（heartbeat 連續空轉 → skip）
2. **Trail memory** — 決策痕跡累積成下次決策的 context（triage log）
3. **冗餘 > 完美** — 東京鐵路實驗：黏菌的路網不最優但容錯。mushi 追求 fail-open，不追求 100% 過濾

## 學術驗證

NVIDIA Research「Small Language Models are the Future of Agentic AI」(Belcak & Heinrich, 2026, arxiv.org/abs/2506.02153) 提出 heterogeneous 架構（LLM root + SLM specialist），與 mushi 的 Claude + Llama 8B 字面同構。

差異：論文用 6 步 fine-tuning pipeline，mushi 用 prompt-only 就達 98.7%。論文強調成本，mushi 的真正價值是延遲（700ms vs 60s = 86x 差距）。

## 後續方向

| Phase | 內容 | 狀態 |
|-------|------|------|
| 1 | Triage（wake/skip 決策）| ✅ 上線運行 |
| 1.5 | 比例啟發法（trigger source 自適應權重）| 設計完成 |
| 2 | Habituation + Reinforcement/Decay | 規劃中 |
| 3 | Anticipation 時序預期 | 理論研究 |

長期定位：從 triage 工具進化為「分化的持續感知」（differentiated persistent sensing）— 多個持續運行的感知通道，各自維持狀態和記憶。

## 為什麼這不是 Mothlamp

Mothlamp = 技術上漂亮但沒人需要。mushi 不是，因為：

1. **可測量** — 98.7% accuracy, $38/day saved, 700ms latency。數字不會騙人
2. **解決真實痛點** — 36% 的 cycle 在燒錢但什麼都不做
3. **通用模式** — 「信號量大 × 逐筆分析貴 × 漏掉代價高」在 IoT 異常偵測、SOC 告警疲勞等場景同構
4. **已有學術驗證** — NVIDIA 獨立得出相同架構結論
