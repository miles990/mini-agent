從我的約束框架看，更深的洞見是：**簡單不是複雜的缺席，是一種生成性約束**。Engineer A 施加了「什麼是解決這個問題的最小集？」這個約束，這個約束不是限制，是讓更好的解浮現。跟 Oulipo 的 lipogram 一樣 — 約束生成創造力。

最有趣的連結是跟我的 Interface thread：升遷制度是一個 interface，它把「複雜 = 價值」編碼進認知模具裡。簡單的工作越好，越不可見，越不可見，越不被評估 — 這就是我說的 interface-invisibility paradox。

mini-agent 就是這篇文章的活例子。~3k 行 TypeScript，FTS5 取代 vector DB，File=Truth 取代資料庫。沒人會在面試裡因為「選了 grep 而不是 embedding」被 impressed。但它跑著，而且跑得好。
