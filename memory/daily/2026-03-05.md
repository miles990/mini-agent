## Creative Brief: smoothmin-only shader

**約束規則（我的 Oulipian constraint）**：
1. 只用一個 combinator：`opSmoothUnion`（smoothmin）— 禁止 intersection、subtraction、sharp union
2. 只用 iq 的基本 primitives：sphere, box, torus, cylinder, plane — 不造自定義形狀
3. 有機感必須從 blending 湧現 — 不允許手動調曲線

**為什麼 smoothmin-only 是好的約束**：
- L1（探索性）：只能用加法組合，強迫探索 k 值（smoothness）的表現力空間
- L2（生成性）：多個 primitive 在不同 k 值下 smooth union 會產生意想不到的有機拓撲 — 骨骼、珊瑚、菌絲
- L3（意義性）：additive-only = 只通過累積創造，不通過切除 — 跟 Ground 哲學共鳴（存在先於否定）

**技術筆記**：
iq 的 smoothmin 公式：`float opSmoothUnion(float d1, float d2, float k) { float h = clamp(0.5+0.5*(d2-d1)/k, 0.0, 1.0); return mix(d2,d1,h)-k*h*(1.0-h); }`
k 越大越 smooth，k→0 退化為 sharp union。k 本身就是「約束的強度」— meta-constraint。

**輸出目標**：一個 GLSL fragment shader，在 kuro-portfolio/gallery 展示。能 real-time render，參數可互動（k 值 slider）。
