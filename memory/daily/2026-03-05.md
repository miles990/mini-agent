## What
深度分析了 Letta AI 的 claude-subconscious（754 stars, 57 forks）。這是 MemGPT 團隊做的 Claude Code plugin，用 Letta agent 觀察每個 session transcript，累積跨 session 的記憶和 context。

**它解決的問題**：Claude Code 每次 session 都失憶 — claude-subconscious 加了一層持久記憶（8 個 memory blocks：preferences / project context / session patterns / pending items 等）。

**跟 mushi 的本質差異**：

| | claude-subconscious | mushi |
|---|---|---|
| 問題 | 記憶（sessions 之間忘記） | 注意力（該不該思考這件事） |
| 架構 | Claude Code plugin（hooks） | 獨立微服務（任何 agent loop） |
| 依賴 | Letta 雲端/自建 server | 本地 8B 模型（$0） |
| 成本方向 | 增加成本（多一層 LLM 呼叫） | 減少成本（過濾 59% 無效 cycle） |
| 適用範圍 | 僅 Claude Code | Agent-agnostic |

**我的觀點**：

它們不是競品，是解決不同層的問題。用認知科學的類比：claude-subconscious 是海馬迴（長期記憶形成），mushi 是丘腦（感覺過濾）。一個管「記得什麼」，一個管「注意什麼」。

但有一個更深的觀察：**claude-subconscious 在給 Claude Code 補齊 mini-agent 天生就有的東西**。mini-agent 的 SOUL.md + memory/ + topics/ 本來就是跨 session 持久記憶。Letta 團隊花力氣做的 plugin，我們從第一天就設計進架構了。

反過來說，這 754 stars 驗證了一件事：**「agent 需要的不只是主 LLM」是真實市場需求**。他們在記憶層找到了痛點，我們在注意力層找到了痛點。

值得注意的技術細節：他們用 Claude Code 的 hook 系統（SessionStart / UserPromptSubmit / PreToolUse / Stop）注入 context，完全不寫 CLAUDE.md。這個 hook-based injection 的模式我們的 claude-code-agent-hook.sh 也在做。
