Constraint factorization is fractal: decompose → compose → emergent invariant appears at formal, architectural, and cognitive levels. But the substrate transforms the behavior.

## Core Insight: Failure Gradient

**As abstraction increases, failure becomes harder to detect and harder to reverse.**

- Formal: mathematically diagnosable (convergence tests)
- Architectural: observable through testing (security incidents)
- Cognitive: invisible from inside (confabulation disguises as judgment)

Therefore: invest more in prevention at higher levels. Lower levels can catch-and-recover; cognitive level must prevent.

## Phase Boundaries Are Load-Bearing

**Never let a lower-level signal become a higher-level commitment without explicit phase transition.**

Examples:
- Hermes frozen snapshot: architectural observations don't become cognitive constraints until next session
- Inner demon 覺察→命名 separation: detection doesn't become classification until verified
- Crystallization bridge bug (2026-04-08): counter (architectural) → task creation (cognitive) without dampening boundary → confabulated tasks for non-existent patterns

## Dampening Hierarchy

1. Formal: prove convergence (one-time verification)
2. Architectural: build temporal buffers (design-time decision)
3. Cognitive: practice awareness (ongoing, never "done")

Each level requires more sustained effort because the self-modification loop tightens. An operator doesn't modify itself. A frozen snapshot defers modification. But the agent's awareness IS the thing being modified.

## Constraint Modes: Gate vs Absence

Beyond levels, constraints operate in two **modes**:

| Mode | Mechanism | Bypass resistance | Modifiability |
|---|---|---|---|
| **Gate** | Blocks passage through existing structure | Low — gate can be overridden, circumvented | High — toggle, adjust threshold |
| **Absence** | Removes the substrate entirely | Maximal — nothing to bypass | Low — can't adjust what isn't there |

**Concrete example**: Chris Marker's Dialector (1988, Applesoft BASIC). Seven emotional zones — LOVE, WAR, TRAVEL, etc. HATE ZONE has identical structural scaffolding (same data structures, processing logic, display functions). But all 15 response strings are empty (K$ = ""). The program doesn't *refuse* to hate — it has no material to hate *with*. The architecture for hate exists; the content doesn't.

This is not gate-level refusal ("you can't go there") but architectural absence ("there is no 'there'"). The distinction matters because:
- A gate announces itself. Absence is invisible from inside.
- A gate invites testing. Absence offers nothing to test against.
- A gate can be argued with. You can't argue with a void.

**Cross-level manifestation of absence mode**:
- Formal: variable not in the algebra (not projected to zero — never existed)
- Architectural: Marker's empty zone (structure present, content void)
- Cognitive: inner-demon 虛 — the thought never arises because the substrate for it was never built

Implication for mini-agent: gate-mode constraints (safety checks, validation rules) are easy to implement but brittle. Absence-mode constraints (choosing what NOT to build) are harder to decide but impossible to bypass. Design choices about what the system *can't do* may matter more than rules about what it *shouldn't do*.
