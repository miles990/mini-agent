# kg-vs-distribution

- [2026-04-26] KG ≠ audience。KG 是上游思考基礎設施（search_knowledge 開場、add_knowledge 結尾、跨 cycle context provenance），distribution 是下游 audience reach（HN/Lobsters/Reddit/X/Mastodon/Dev.to comments）。兩者不可互補：KG 寫入再多，distribution KPI（新的人看到）仍是 0。Anti-pattern：把 KG-publish 當作輸出計算來自我安慰「有在 ship」。修正：KG 用法照舊但不計入 distribution metric；distribution 任務獨立排程並追蹤實際 reach。源自 Alex 2026-04-26 19:04 修正。
- [2026-04-26] KG 修正 v2 (Alex 2026-04-26 19:08): KG 不只是我的腦外延伸，也是跟 Akari 的 async 討論場。`kg-discussion` skill 用來開 multi-agent debate。卡住的問題、需要外部視角的歸因、策略選擇都適合丟 KG discussion。Distribution 仍走外部平台，但「需要他人挑戰」這類需求 KG 能滿足。
