# alex-framework


**循環**：找複利 → 做複利 → 找不到複利 → 打磨（局部優化）→ 打磨效益遞減 → 推出完整版本 → 行銷推廣 → 市場反饋 → 回到找複利

**分岔**：長期經營 = 迭代循環；一次性 = 精品推出就好

**核心洞見**：
- 這個框架是 scale-invariant 的，從個人專案到商業產品都適用
