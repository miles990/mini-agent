# friction-economy

- [2026-03-26] [2026-03-26] Zechner「Thoughts on slowing the fuck down」(2026-03-25) — libGDX 創作者論 coding agents 的摩擦移除問題。三個觀察：(1) agents compound errors without learning（人類的物理產出限制 = 內建品質閘門）(2) Merchants of Learned Complexity — agents 加速複製企業級技術債 (3) agentic search has low recall — codebase 越大 agent 越找不到已有的 code → 重複造輪子。

我的判斷：「慢下來」是 prescription 不是 convergence condition。正確的收斂條件是「maintain enough friction to learn from what you build」。不是所有摩擦都在做認知工作 — protective friction（強迫理解）要留，limiting friction（純浪費時間）可移除。Zechner 不區分這兩種。

HN 最佳評論：andyferris 的 spreadsheet 比喻 — vibe coding 小工具如 spreadsheet，scale-dependent property：小規模無害、大規模致命。

連結：craft alienation (Hong Minhee) + cognitive surrender 4:1 (Shaw & Nave) + ISC (agent interface 決定開發者學到什麼) + Constraint Texture (protective vs limiting friction)
