## Patch 3c: `graph.html` — renderer template edit

graph.html is fully overwritten on every nightly cron run. **Direct HTML edits will be lost.** Must edit the template in:

```
mini-agent/scripts/hn-ai-trend-graph.mjs
```

### 3c.1 Strategy

Find the template literal (or HTML string concatenation) inside `hn-ai-trend-graph.mjs` that emits the `<body>` / `<header>` of graph.html. Insert the same onboarding banner pattern as base-patch (swimlane), keyed `kuro.ai-trend.onboard.graph.v1`.

### 3c.2 Banner copy for graph view

> 每個圓點 = 一篇 AI 相關貼文。**顏色** = 來源（HN 黃 / Reddit 紅 / X 黑 / arXiv 綠 / Latent 藍）。**距離** = 主題語義相近度。<kbd>drag</kbd> 移動、<kbd>scroll</kbd> 縮放、<kbd>click</kbd> 看原文。

### 3c.3 Apply order

1. Read `scripts/hn-ai-trend-graph.mjs` to locate template insertion point (look for `<body>` opening or first markup string after style block).
2. Insert CSS block (same as swimlane base patch, adapt selector if conflicts).
3. Insert markup block + dismiss script with localStorage key `kuro.ai-trend.onboard.graph.v1`.
4. Run renderer: `node scripts/hn-ai-trend-graph.mjs` → regenerates `kuro-portfolio/ai-trend/graph.html` with banner baked in.
5. Verify: open file, grep for `id="onboard"` → should appear.

### 3c.4 Falsifier for 3c

If after applying 3c and re-running the renderer, the resulting graph.html has no `id="onboard"` element, the template insertion point was wrong (likely missed a string concatenation or the template uses `${}` interpolation that ate the markup). Re-locate insertion point; do not re-apply blindly.
