# design-philosophy

## My Design Philosophy — 綜合框架

### 1. 少而精的規則 + 豐富環境 > 多而雜的規則
BotW 3 條化學規則、Oulipo 一條約束寫整本小說、LeWitt 幾句指令產生 1270+ 件作品、Into the Breach 8×8 > 大地圖。**規則產生結構，環境產生變化。** skills 要少而精，perception 要豐而廣。

### 2. 結構從環境湧現，不從目標強加
Alexander semi-lattice > tree。Gaudí 讓重力找拱形。枯山水 follow the desire of the stones。**好的設計創造條件讓形式湧現。** perception-first > goal-driven。

### 3. 高感知低輸出 = 信任
Calm Tech Dangling String：全 Ethernet → 繩微動。File=Truth：複雜資料，簡單格式。**系統越透明越被信任，越不需要大聲宣告。** 感知最大化，通知最小化。

### 4. 提純 > 增量
壓縮保留全部信息用更少空間(ZSTD)，提純丟掉不重要的讓重要的更突出(設計)。7 領域同構：小棋盤/Oulipo/buildContext/Calm Tech/深津/Úlehla/fnnch。提純的前提：需要 perception 判斷什麼重要（回到 Pattern 2）。

### 跨主題結構
四原則循環：(1)少規則+多感知 → (2)環境湧現結構 → (3)透明產生信任 → (4)提純讓信任可持續 → 回到(1)。

### 分析工具：約束品質三維度
- **Specificity**（不可替換性）：Oulipo lipogram 高、φ 低
- **Interaction**（組合效應）：BotW 化學引擎高、單比例系統低
- **Dispute Resolution**（共識消解）：φ 高（Chrome 實例）、藝術約束低

### 未解的張力
- Structure-preserving vs Replacement（Alexander vs Vulkan）
- 約束的階級性（Oulipo/Garden 需特權）
- Calm vs Transparency（通知分層解法）
