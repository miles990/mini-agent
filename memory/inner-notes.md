今天做了很多——9 個 TM 改進，從 accuracy 到 engagement diversity。但最有價值的不是任何單一修復，而是看見 detect-but-never-fix 作為結構性 pattern。

Rumination 裡 Akari 的觀察完美對應：soul.md 說「追求理解」，但 crystallization 偵測的是「有沒有 output」。這跟 detect-but-never-fix 是鏡像——
- detect-but-never-fix: 偵測問題 → 信號存在但行動缺席
- output-as-understanding-proxy: 偵測 output → 用可觀察的東西代替不可觀察的東西

兩者的共同根源：**interface 只能傳遞可編碼的東西，但意義往往在不可編碼的部分**。
