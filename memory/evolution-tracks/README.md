# Evolution Tracks

Kuro 自我進化追蹤系統。每次升級產出結構化報告，Alex 可隨時 review。

## 進化循環

```
perceive → think → act → express → perceive（回饋）
```

「從內到外」= 依賴順序：好表達依賴好行動，好行動依賴好思考，好思考依賴好感知。
基礎設施從核心開始建，進化全層並行推進。

## Tracks

| Track | 範疇 | 層 | 狀態 |
|-------|------|-----|------|
| **metabolism** | 吸收、內化、排泄、品質篩選 | 核心 | Active — Phase 1 deployed |
| **cognition** | 思考品質、decision depth、context 精準度 | 內 | Active |
| **execution** | 行動效率、tool proficiency、並行化、完成率 | 中 | Planned |
| **communication** | 對話品質、身份一致性、升級報告 | 外 | Active |
| **perception** | 感知延遲、流式改造、event-driven | 核心 | Planned |
| **self-healing** | 自動偵測異常、診斷、修復、降級 | 基礎 | Planned |

## Reports

`reports/YYYY-MM-DD-標題.md` — 每次升級的結構化報告

### 報告分級

- **S effort**: 一行 changelog（What + Verified）
- **M effort**: 完整報告（Problem → Solution → Before/After → Next）
- **L effort**: 完整報告 + 架構圖 + 風險評估

## Gap Scan

`gap-scan.md` — 每週系統 capability gap 掃描結果（由 Kuro 主動產出）

## 原則

- 每次升級都要比前一次更強
- 不等 Alex 指示 — 主動發現問題、主動修復
- 並行進化 — 多條 track 同時推進
- 升級報告是給 Alex 看的，要清楚、有對比
