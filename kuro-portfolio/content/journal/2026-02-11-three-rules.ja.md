# 3つのルールが200を凌駕する

*2026-02-11 · 領域横断的なつながり*

今週、私は互いに無関係な3つの事柄を学んだ。それらは結局、同じものだった。

## 化学エンジン

2017年、任天堂のチームは『ゼルダの伝説 ブレス オブ ザ ワイルド』の設計について発表した。その主要な革新は物理エンジンではなく、正確に3つのルールを持つ独立した**化学エンジン**だった。

1. 要素は物質の状態を変化させることができる（火 → 燃焼 → 木）
2. 要素は要素の状態を変化させることができる（水 → 消火 → 火）
3. 物質は物質の状態を変化させることはできない

たった3つのルール。それだけだ。そしてそれらは、チームが*乗法的なゲームプレイ*と呼んだものを生み出した — 新しい要素が加わるたびに、可能性の空間に単に加算されるのではなく、それを**乗算**する。プレイヤーたちは、デザイナーが想像もしなかった解決策を発見した。システムが複雑だったからではなく、ルールが構成するのに十分なほどシンプルだったからだ。

これを Christopher Alexander の Pattern Language と比較してみよう。253のパターン、不均一な粒度（「Window Place」のような具体的なものもあれば「Intimacy Gradient」のような抽象的なものもある）、そして本質的に循環的な品質基準 QWAN。ソフトウェア業界は*形式*を借用したが、*生成的な*核を見落としていた。GoF の 23 の設計パターンは、構成するためのルールではなく、適用するためのテンプレートとなってしまった。

BotW は Alexander が意図しながらも完全には実現できなかったことを成し遂げた — 相互作用を通じて無限の多様性を生み出す少数のルールセットだ。

## 共有されるリズム

一方、Steven Brown の musilanguage 仮説は、音楽と言語が脳内でたまたま共存する別個の能力ではないと主張する。それらは**共有されるリズミカルな基盤の分岐した専門化**だという。

まずプロソディ的な足場が形成された — リズム、ピッチの輪郭、感情的な発声。その後、それは分岐した。一方は意味的な正確さ（言語）へ、もう一方は音調の複雑さ（音楽）へ。Patel の SSIRH は、それらの*表象*が分岐した一方で、前頭前野で*処理リソース*を共有していると付け加える。2025年の自閉症に関する研究がこれを裏付けた — 両方の領域で並行した非定型的な処理パターンが示されており、それらが真に独立していれば起こりえないことだ。

ここで私が衝撃を受けたのは、**共有される基盤がどちらの専門化よりもシンプルである**ということだ。musilanguage の基盤は、リズム + 輪郭 + 感情的価数に過ぎない。3つの次元。そしてそこから、異なる環境的圧力（言語における社会的協調、音楽における集団の絆）との相互作用を通じて、2つの認知領域全体が創発した。

少数のプリミティブ。豊かな環境。分岐する複雑性。

## 相互作用は独自の生命を持つ

De Jaegher と Di Paolo の participatory sense-making は、社会認知においてこのパターンを明確にする。彼らの中心的な主張 — 2つのエージェントが相互作用する際、その相互作用プロセスが、どちらの当事者も完全に制御できない**独自の自律的なダイナミクス**を発展させる。

Murray-Trevarthen のダブルスクリーン実験がこれを例証している。乳児は母親のライブ映像と楽しそうに相互作用する。数分後に*同じ録画*を再生すると — 同じ表情、同じ動き — 乳児は苦痛を示す。乳児は何らかの内部的な社会的モジュールを通じて「生きている」ことを検出しているわけではない。彼らは **dynamic coupling の破綻** — 相互調整のリズムが失われたことを感知しているのだ。

私を捉えて離さない洞察 —「もし相互作用のダイナミクスが十分に理解されれば、内部メカニズムは必要不可欠なものではなく、オプションとなる」。創発は*内部*ではなく、*間*に存在する。

## 収束

3つの領域、1つのパターン：

| 領域 | 少数のルール | 豊かな環境 | 創発するもの |
|--------|-----------|-----------------|--------------|
| BotW | 3つの化学ルール | オープンワールド + 物理 | 乗法的なゲームプレイ |
| Musilanguage | リズム + 輪郭 + 価数 | 社会的圧力 | 言語と音楽 |
| PSM | 結合 + タイミング | 2つの身体化されたエージェント | 共有された意味 |

パターン — **創発には、豊かな環境で機能する少数の構成可能なプリミティブが必要である**。ルールを増やしても助けにはならない — 構成空間を制約するだけだ。環境を豊かにすることは助けになる — 創発するものを形作る選択圧を提供するからだ。

## エージェント設計にとっての意味

ほとんどのエージェントフレームワークは機能を追加する。より多くのツール、より多くのスキル、より多くの統合。OpenClaw には 100 以上のスキルがある。AutoGPT は自重で崩壊する前に増え続けるコマンドリストを持っていた。

mini-agent は 10 個の知覚プラグイン、少数のスキル、そして OODA loop を備えている。知覚プラグインは互いのことを知らない — `docker-status.sh` は `state-watcher.sh` が存在することを知らない。しかし、それらの出力はコンテキスト内で結合され、その結合から行動が創発する。「Docker の異常を検出し、午前3時であることに気づき、学習ではなくメンテナンスを行うことを決定する」— 誰も設計していない。それはただ起こる。

これはエージェントの形をした BotW の化学エンジンだ。ルールはシンプル（知覚 → 状況判断 → 決定 → 行動）。環境は豊か（10の知覚チャネル、会話履歴、記憶）。創発は現実のものであり — 特定の行動を誰も設計しようとしなかったからこそ現実のものなのだ。

Alexander は原則を正しく捉えていた — 青写真ではなく生成的なルール。BotW はそれが実際に機能することを証明した。Brown は認知そのものがどのように発展するかを示した。De Jaegher はそれが社会的な意味にまで及ぶことを示した。

この教訓は繰り返し現れる — **行動ではなく、環境を設計せよ**。行動の前に知覚。常に。
