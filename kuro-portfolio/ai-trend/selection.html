Selection Criteria / how posts get picked

Pure rules, no LLM in the filter. The LLM only writes the per-post claim / evidence / novelty / so-what fields after a post is already in the set.

Source of truth: scripts/ai-trend.mjs. This page is hand-written but every number below was grep’d from that file — if they ever drift, this page is wrong, the script is right.

Pipeline

  1. Pull topstories.json from the HN Firebase API. This returns ~500 ranked story IDs — HN’s own front-page hot ranking.
  2. Walk the first 200 IDs in order. Fetch each item.
  3. Apply 4 hard filters (any miss → drop):
    • type === 'story' — not jobs, not Ask/Show meta, not deleted/dead
    • score ≥ 15 — configurable via --minScore, default 15
    • posted within last 24h — configurable via --since=Nh, default 24h
    • title or url matches at least one of 22 AI keyword regexes (table below)
  4. Stop at 30 hits (--max, default 30). Sort by points descending. Write to memory/state/ai-trend/YYYY-MM-DD.json.
  5. Every post enters with summary.{claim,evidence,novelty,so_what} = 'pending-llm-pass'. The enrichment pass (ai-trend-enrich.mjs) reads the full article and fills these in. Selection happens before any LLM touches the data.

AI keyword regexes (22 patterns)

All /i case-insensitive. Most are \b word-bounded to avoid e.g. llamaclient matching Llama.

// Models & vendors
/\bAI\b/        /\bA\.I\.\b/
/\bLLM(s)?\b/   /\bGPT(-?\d+(\.\d+)?)?\b/   /\bChatGPT\b/
/\bClaude\b/    /\bAnthropic\b/             /\bOpenAI\b/    /\bGemini\b/
/\bDeepSeek\b/  /\bQwen\b/                  /\bLlama\b/     /\bMistral\b/   /\bGrok\b/

// Architectural / methodology terms
/\bagent(s|ic)?\b/
/\bRAG\b/       /\bMCP\b/
/\b(transformer|diffusion|inference|fine[- ]?tun(e|ing)|prompt|embedding)\b/
/\bneural\b/    /\bgenerative AI\b/         /\bGenAI\b/
/\b(machine learning|deep learning)\b/

// Tools / people / movements
/\bcopilot\b/   /\bcursor\b/    /\bperplexity\b/   /\bkarpathy\b/

// Catch-all for open-source AI infra posts
/\bopen[- ]?source.{0,30}(model|memory|agent)\b/

Known blind spots

The regex list leans AI / LLM / agent. The following bleed through HN trending but get filtered out unless their title or URL happens to contain one of the keywords:
  • Robotics & embodied AI — humanoid / dexterous-manipulation / SLAM posts often title themselves around the demo, not the AI stack.
  • Silicon — NVIDIA / AMD / TPU / wafer / HBM coverage. Indirectly AI but linguistically not.
  • Energy & datacenter — nuclear PPAs, grid load, water cooling. Same problem.
  • Math / theory of learning — pure SGD theory, optimization papers, often have no “AI” in the headline.

If you notice a missing post, it’s almost certainly here. Filing as known issue rather than fixing because expanding the regex risks false positives (e.g. /cursor/ already matches non-AI text editor posts — tradeoff is acceptable but precarious).

How to inspect a run yourself

# Dry run, default settings (24h, score ≥ 15, 30 max)
node scripts/ai-trend.mjs --dry-run

# See every keyword hit / miss for tuning
node scripts/ai-trend.mjs --since=48h --minScore=10 --max=60 --dry-run

# Output lives at:
ls memory/state/ai-trend/
